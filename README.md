### STEP 1. 주제 설정 및 데이터 선정

> 💡 **진행 팁:** 큰 주제 → 데이터 탐색 → 주제 구체화 순서로 진행하세요. 주제 선정에 너무 많은 시간을 쓰지 않는 것이 핵심입니다.

#### 1-1. 큰 주제 브레인스토밍
- [ ] 팀원 각자 관심 도메인을 슬랙에 자유롭게 포스팅
- [ ] 슬랙 이모지 반응으로 1차 투표 → 큰 주제 1~3개로 압축
- [ ] 줌 회의에서 주제 후보 토론 및 최종 방향 결정
- [ ] 결정된 주제 후보와 근거를 슬랙에 정리하여 공유
- [ ] 해당 주제의 공개 데이터 존재 여부 확인 (크롤링, 공공데이터포털, Kaggle, AI Hub 등)

#### 1-2. 좋은 주제 vs 나쁜 주제 기준 확인
- [ ] 주제가 좁고 구체적인 수준(예: 특정 대상, 특정 행동, 명확한 영향 요인 분석 등)으로 좁혀졌는지 확인
- [ ] 줌 회의에서 팀원들이 주제를 듣고 "무엇을 분석하는지" 즉시 이해하는지 확인

#### 1-3. 데이터 선정
- [ ] 팀원 각자 찾은 데이터셋을 슬랙에 링크 및 간략한 설명과 함께 공유
- [ ] 줌 회의에서 데이터셋 검토 및 최종 선정
- [ ] 선정된 데이터 명세를 슬랙에 정리하여 공유
  - 출처 및 원본 링크, 파일 형태, 수집 기간, 행/열 수, 주요 컬럼 설명
- [ ] 데이터 라이선스·저작권 확인 후 슬랙에 공유
- [ ] 원본 데이터 파일을 팀 공유 폴더(Google Drive 또는 GitHub)에 저장 후 슬랙과 노션에 링크 공유

#### 1-4. 초기 주제 가설 수립
- [ ] 슬랙에서 가설 후보 자유 제안 및 의견 교환
- [ ] 줌 회의에서 가설 검토 및 최종 2~5개 확정
- [ ] 확정된 주제와 가설을 슬랙과 노션에 기록
- [ ] 줌 회의 요약을 슬랙과 노션에 공유

### STEP 2. 기본 EDA (탐색적 데이터 분석)

> 💡 **진행 팁:** 그래프와 교차표를 최대한 많이 그려보세요. 각 유형별 최소 10개 이상이 목표입니다. 각 그래프마다 인사이트 코멘트를 반드시 작성하세요.

#### 2-1. 데이터 전처리
- [ ] 결측값 현황 파악 및 처리 방식 결정 — 제거 / 대체(평균·중앙값·최빈값) / 유지
- [ ] 중복 데이터 확인 및 제거
- [ ] 데이터 타입 변환 — 날짜 컬럼 → datetime, 숫자형 컬럼 정비
- [ ] 이상값(Outlier) 탐지 — 박스플롯, IQR 기준으로 확인
- [ ] 파생 변수 생성 (필요 시) — 연령대, 매출 구간, 요일 등
- [ ] 전처리 기준과 결과를 슬랙에 공유
- [ ] 전처리 방식에 이견이 있으면 줌 미니 회의 또는 슬랙 스레드로 빠르게 합의

#### 2-2. 단변량 분석 (10개 이상)
- [ ] 수치형 변수 히스토그램 — 분포, 정규성 여부, 편향 방향
- [ ] 수치형 변수 박스플롯 — 이상값 및 분포 범위
- [ ] 범주형 변수 막대그래프 — 카테고리별 빈도
- [ ] 기술통계 요약표 — mean, median, std, min, max, 분위수
- [ ] 시계열 데이터의 경우 시간별 추이 그래프
- [ ] 분석 중 막히는 부분은 슬랙에 스크린샷 + 코드 스니펫 공유

#### 2-3. 다변량 분석 (10개 이상)
- [ ] 수치형 변수 상관관계 히트맵 — 상관계수 0.5 이상인 쌍 메모
- [ ] 범주형 × 수치형 교차 박스플롯
- [ ] 범주형 × 범주형 교차표(Crosstab) — 비율·건수 모두 확인
- [ ] 피벗 테이블 활용 그룹별 집계
- [ ] 산점도 — 주요 수치형 변수 쌍 조합
- [ ] 각 그래프에 인사이트 코멘트 1줄 이상 기재

#### 2-4. 분석 코멘트 작성
- [ ] 그래프의 단순 설명을 넘어 **수치적 비교, 원인 해석, 비즈니스적 시사점**이 포함된 좋은 코멘트 작성 여부 확인

#### 2-5. EDA 결과 정리
- [ ] 주제와 연관성 높은 분석 결과 상위 5개 선별
- [ ] 예상치 못한 인사이트 슬랙에 공유 — 팀원 의견 수렴
- [ ] 추가로 필요한 데이터 목록 슬랙에 공유
- [ ] EDA 결과를 줌 회의에서 팀 전체에 발표 및 피드백 수렴
- [ ] 회의 후 주요 피드백과 다음 방향을 슬랙에 요약 공유

### STEP 3. 주제 확정 및 심화 분석

> 💡 **진행 팁:** EDA 결과를 바탕으로 주제를 좁혀 잡으세요. 주제는 좁을수록, 검증 가능할수록 좋습니다.

#### 3-1. 주제 확정
- [ ] EDA 인사이트 기반으로 주제 재검토 — 기존 가설과 비교, 보정 또는 교체
- [ ] 주제 범위 구체화 및 분석 목적·대상·기간 명확화
- [ ] 줌 회의에서 팀 전원 합의 후 최종 주제 확정
- [ ] 확정된 주제를 슬랙에 공지

#### 3-2. 심화 분석
- [ ] 핵심 질문 3~5개를 슬랙에 공유하며 팀원 의견 수렴
- [ ] 그룹별 비교 분석 — 세그멘테이션 기준 정의 후 집단 간 차이 검증
- [ ] 시계열 분석 (해당 시) — 추세·계절성·이상 구간 탐지
- [ ] 상관/회귀 분석 (필요 시) — 변수 간 관계 수치화
- [ ] 각 인사이트에 비즈니스 의미 해석 추가
- [ ] 비즈니스 액션 아이디어 초안 슬랙에 공유
- [ ] 심화 분석 결과를 줌 회의에서 발표 및 팀 피드백 수렴

- [ ] 주요 인사이트와 가설 검증 결과를 슬랙에 누적 정리
- [ ] 주차별 진행 현황을 슬랙에 업데이트

### STEP 5. 발표 준비 및 최종 점검

#### 5-1. 슬라이드 구성 원칙
- [ ] 간지(섹션 구분 슬라이드) 삽입 — 챕터가 바뀔 때마다 흐름 구분
- [ ] 슬라이드당 텍스트 최소화 — 핵심 키워드·수치 중심, 문장 나열 금지
- [ ] 그래프는 제목 + 핵심 수치 + 한 줄 해석 함께 배치
- [ ] 분석 결과 그래프는 필요한 만큼 포함 — 슬라이드당 그래프 1~2개 권장
- [ ] 강조할 수치·키워드는 크게, 부가 설명은 작게 — 시각적 위계 구성

#### 5-2. 데이터 신뢰도 슬라이드
- [ ] **데이터 개요 슬라이드** 작성 (출처, 수집 기간, 대상, 레코드/컬럼 수, 주요 컬럼 설명)
- [ ] **전처리 요약 슬라이드** 작성 (결측치/이상치 처리 방법 및 비율, 최종 데이터 수, 파생 변수 내역)
- [ ] 데이터 한계점 1~2줄 언급

#### 5-3. 발표 자료 스토리라인 점검
- [ ] 표지 (팀명, 주제, 날짜, 팀원) 및 목차 슬라이드
- [ ] **[간지] 문제 정의** (분석 배경 및 목적)
- [ ] **[간지] 데이터 소개** (개요 및 전처리 요약 슬라이드)
- [ ] **[간지] 분석 결과** (질문별 그래프 + 해석)
- [ ] **[간지] 결론 및 비즈니스 액션플랜**
  - 인사이트 요약 (핵심 발견 3~5개)
  - 비즈니스 액션플랜 (인사이트 → 제안 액션 → 기대 효과(KPI), 단기/중장기 구분, 실행 주체 명시)
- [ ] 마무리 슬라이드 — Q&A 안내

#### 5-4. 비즈니스 인사이트 및 액션플랜 품질 검토
- [ ] 각 인사이트가 단순 사실 나열이 아닌 원인과 방향성을 포함하는지 팀원 간 교차 검토
- [ ] 액션플랜에 **인사이트 근거 + 실행 주체 + 기대 KPI** 세 가지가 모두 포함되었는지 확인

#### 5-5. 슬라이드 협업 완성 및 최종 점검
- [ ] 슬라이드 초안을 슬랙에 공유하며 팀원 피드백 수렴
- [ ] 줌 회의에서 슬라이드 리뷰 및 발표 흐름 점검
- [ ] 피드백 반영 후 최종본 슬랙에 공유
- [ ] 간지 슬라이드 누락 여부 확인
- [ ] 데이터 개요·전처리 수치가 실제 데이터와 일치하는지 검증
- [ ] 비즈니스 액션플랜이 분석 결과와 논리적으로 연결되는지 확인
- [ ] 줌에서 발표 리허설 진행 (시간 체크)
- [ ] Q&A 예상 질문 및 답변 슬랙에 정리
- [ ] 제출물 최종 파일 목록 슬랙과 노션에 공유 — 코드, 데이터, 발표자료 링크
