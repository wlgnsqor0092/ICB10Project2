# 📊 네이버 API 통합 분석 대시보드 개선 제안서

본 문서는 네이버 API 데이터 통합 분석 대시보드의 성능, 분석 기능 및 사용자 편의성을 높이기 위한 고도화 개발 제안 사항을 정리한 문서입니다.

---

## 🛠️ 1. API 호출 성능 최적화 (비동기 처리)
현재 대시보드는 여러 API(블로그, 카페, 뉴스, 쇼핑) 데이터를 순차적(동기식)으로 요청하고 있어, 수집 데이터 건수가 늘어날 때 응답 지연이 발생합니다.

*   **개선 방안**:
    - `requests` 라이브러리 대신 비동기 HTTP 클라이언트인 `httpx` 또는 `aiohttp` 도입.
    - `asyncio.gather()`를 활용해 블로그, 카페, 뉴스, 쇼핑 데이터를 **병렬로 동시 수집**하도록 [naver_api.py](file:///Users/jihoon/Desktop/ICB10Project2/naver-api-app/src/naver_api.py) 코드 리팩토링.
*   **기대 효과**: 데이터 로딩 속도 최대 **3~4배 단축**.

---

## 🔍 2. 시각화 및 데이터 분석 고도화
단순히 수집된 데이터를 나열하고 기본적인 가격 분포를 보여주는 수준을 넘어 깊이 있는 인사이트를 제공합니다.

*   **워드클라우드(Word Cloud) 및 연관어 분석**:
    - 수집된 블로그/뉴스 제목 및 본문 텍스트에서 명사 빈도를 분석하여 중요 키워드 시각화.
*   **텍스트 감성 분석(Sentiment Analysis)**:
    - 수집된 피드의 제목에 긍정/부정 사전을 매핑하거나 규칙 기반 감성 점수를 계산하여 **여론 추이 파이차트** 제공.
*   **기대 효과**: 정량 데이터와 함께 정성 데이터(텍스트) 분석을 결합하여 입체적인 시장 조사 지원.

---

## 💾 3. 보고서 다운로드 및 편의 기능 추가
분석 결과 데이터를 실무 보고에 즉시 활용할 수 있도록 편의성을 강화합니다.

*   **엑셀(Excel) 및 PDF 리포트 내보내기**:
    - 분석된 KPI 지표, 차트 이미지, 데이터 원본을 깔끔하게 정리된 Excel 파일(`openpyxl` 서식 적용) 또는 PDF로 다운로드하는 기능 추가.
*   **검색 조건 즐겨찾기(Bookmarks)**:
    - 자주 분석하는 키워드 조합(예: "네이버,카카오,구글")과 기간, 옵션 설정을 세션이 종료되어도 유지 및 불러올 수 있는 즐겨찾기 목록 구현.
*   **기대 효과**: 대시보드 사용성 향상 및 실무 보고서 작성 시간 단축.
