# 📊 네이버 API 통합 분석 대시보드 개선 계획서

본 문서는 사용자의 요구사항을 반영하여 네이버 API 통합 분석 대시보드의 다중 검색어 종합 분석 화면 개발 계획을 정리하는 문서입니다.

---

## 💡 주요 개발 목표
1. **다중 검색어 일괄 분석**: 콤마(`,`)로 구분된 여러 검색어를 한 번에 입력받아 분석합니다.
2. **뉴스, 블로그, 카페 통합 수집**: 여러 검색어 각각에 대해 뉴스, 블로그, 카페 데이터를 병렬로 동시 수집합니다.
3. **가벼운 텍스트 분석**: 무거운 형태소 분석기(KoNLPy 등) 대신 가벼운 문자열 파싱 및 정규식 매칭을 이용한 단어 빈도 분석을 수행합니다.
4. **종합 분석 리포트 화면**: 각 플랫폼 데이터를 결합하여 전체 트렌드와 플랫폼별(뉴스 vs 블로그 vs 카페) 분포를 통합 시각화합니다.

---

## 📅 설계 의사결정 기록 (인터뷰 완료)

### Q1. 다중 검색어 통합 시각화 방식
- **결정 사항**: **'종합 분석' 화면 신설**
- **세부 내용**: 
  - 콤마로 구분해 입력한 모든 검색어에 대하여 뉴스·블로그·카페의 총 발행량 합산 추이를 시계열로 나타내고, 플랫폼 간 점유율(뉴스 vs 블로그 vs 카페) 차트를 원형 그래프(Pie Chart)로 제공합니다.
  - 종합 지표 아래에 각 검색어별 상세 비교 차트를 구성하여 검색어 간 대조가 쉽도록 유도합니다.

### Q2. 가벼운 텍스트 분석 기법 세부 설계
- **결정 사항**: **정규식 및 내장 불용어 필터링 기반 분석**
- **세부 내용**:
  - 형태소 분석기 없이 가볍고 빠른 처리를 위해, 공백 단위로 단어를 분할한 후 2글자 이상인 단어만 필터링합니다.
  - 자주 쓰이는 불용어(조사, 지시어 등) 리스트를 모듈 내에 내장하여 이를 걸러낸 후 최종 빈도수 Top 10 키워드 리스트를 산출합니다.

### Q3. 데이터 수집 및 병렬화 범위
- **결정 사항**: **ThreadPoolExecutor를 이용한 병렬 수집 및 수집 건수 최적화**
- **세부 내용**:
  - 검색어 개수가 늘어남에 따라 API 요청 횟수가 기하급수적으로 늘어나는 점을 방지하기 위해 검색어당/채널당 수집 건수를 30~50건 수준으로 최적화합니다.
  - 동기 호출에 따른 지연을 극복하기 위해 `concurrent.futures.ThreadPoolExecutor`를 활용하여 뉴스, 블로그, 카페 검색 API를 동시에 병렬 호출하고 결과를 취합합니다.
